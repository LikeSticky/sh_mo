General Formatting Rules

  • 공백문자는 하드탭이 아닌 소프트탭으로 스페이스4개를 적용(하드탭은 기본공백이 에디터마다 상이하여, 동일렌더링을 보장하는 소프트탭을 사용)
  • 들여쓰기는 하드탭으로 변경 1탭 4스페이스로 적용은 탭으로 적용
  • 가독성을 높이기 위해 여는 중괄호 앞에 하나의 공백 문자를 삽입
  • 각각의 선언에서 콜론(:)뒤에는 공백 문자
  • 다중 속성값의 구분을 위해서만 콤마뒤에 공백 문자 혹은 줄바꿈 처리 (ex: box-shadow)하며 속성값 안에서는 콤마구분에 공백삭제 (ex:background-color)
  • 속성값들과 색상값들에 소숫점이 필요할 경우 0은 사용하지 않음
  • 16진수 값들은 소문자로 표현하며 축약형태로 사용 (예: #acacac, #333)
  • 선택자 안에서는 속성값 및 백그라운드 속성은 큰따옴표를 사용 (예: input[type="text"])
  • 0값들에서는 단위생략. (예: margin: 0;)
  • 폰트의 기본 단위는 px을 사용하며, %, vw, vh은 선택적으로 사용가능

code pattern

                                    .selector[type='text'],
                                    .selector-secondary {
                                    padding: 15px;
                                    margin-bottom: 15px;
                                    background-color: rgba(0, 0, 0, .5);
                                    box-shadow: 0 1px 2px #ccc, inset 0 1px 0 #fff;
                                    }

속성 선언 순서[선택사항]

  • 속성에 선언순서는 포지션 관련 > 박스 모델 > 타이포그라피 > 시각 효과
포지션 관련 선언들이 가장 먼저 옵니다. 문서의 정상적인 흐름에서 엘리먼트들을 없앨 수 있고, 박스 모델 관련 스타일들을 재정의할 수 있기 때문입니다.
박스 모델이 그 다음으로 오는데, 이것은 해당 엘리먼트의 크기와 위치를 지정하기 때문입니다.
나머지는 이 앞의 두 섹션에 영향을 주지 않거나 해당 엘리먼트의 내부에서만 일어나기 때문에 나중에 배치 시킵니다.

                                    .declaration-order {
                                    /* Positioning */
                                    position: absolute;
                                    top: 0;
                                    right: 0;
                                    bottom: 0;
                                    left: 0;
                                    z-index: 100;

                                    /* Box-model */
                                    display: block;
                                    float: right;
                                    width: 100px;
                                    height: 100px;

                                    /* Typography */
                                    font: normal 14px artial, sans-serif;
                                    line-height: 1.3;
                                    color: #333;
                                    text-align: center;

                                    /* Visual */
                                    background-color: #f5f5f5;
                                    border: 1px solid #e5e5e5;
                                    border-radius: 3px;

                                    /* Misc [miscellaneous - 여러가지 잡다한]*/
                                    opacity: 1;

@import 사용 금지 가이드시에 import사용중 최종결정 필요

<link>에 비해 @import는 상대적으로 느리고 추가 페이지 요청을 필요로 하기 때문에 로딩관련하여 예측할 수 없는 문제를 야기하는 경우가 있어 @import의 사용을 피하고 <link>를 사용

code pattern

                                    
                                    <link rel="stylesheet" href="/contents.css">

Class Naming

  • 클래스에서 자연스러운 단어별 브레이크(Ctrl+방향키)를 제공을 위해 클래스 이름은 소문자와 하이픈(-)을 사용
  • 의미없는 클래스명 보다는 명시적으로 선언
  • 클래스 이름은 축약형태로 선언하며, 유추하기 어려운 축약은 사용자제
  • 의미 있는 이름을 사용하여 구조와 목적을 나타내는 네이밍으로 표현
  • prefix는 베이스나 가장 가까운 부모를 바탕으로 네이밍

code pattern

                                    .keyvi-intro {border: 1px solid #000;}
                                    .foot-menu-complaints {border: 1px solid #000;}

구성

  • 콤포넌트 및 모듈 그룹에 맞춰 코드를 구성
  • 일관적인 상하관계에 맞추어 개발
  • 페이지 대신 콤포넌트 및 모듈별로 그룹핑

에디터 설정

  • 파일 저장시 utf-8 인코딩으로 저장
  • 하드탭 설정을 공백 문자 네개로 구성된 1tab으로 세팅
  • 저장시 파일 마지막에 오는 불필요한 공백문자들은 제거(bom+제거)
